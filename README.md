# Claude_CPP_VideoPlayer

Using Claude AI (Opus), building video players in CPP as a test bed. The majority of this code (including the README) will be generated by Claude.

---

## Prerequisites

Before building, make sure the following are installed on your Windows machine:

- **Visual Studio 2022** with the **"Desktop development with C++"** workload
- **CMake 3.20+** — [cmake.org/download](https://cmake.org/download/) (add to PATH during install)
- **Git LFS** — [git-lfs.github.com](https://git-lfs.github.com/) (required to pull the bundled `.dll`, `.lib`, and `.mp4` files)

The FFmpeg and SDL3 libraries are bundled in the `libraries/<OS_TARGET>/` directories and tracked via Git LFS, so no separate installation is needed.

---

## Cloning

```bash
git lfs install
git clone https://github.com/dougp7445/Claude_CPP_VideoPlayer.git "C++ Video Player"
cd "C++ Video Player"
```

If you cloned before installing Git LFS, run `git lfs pull` to fetch the binary files.

## For Windows

### Building

Run the build script from the repo root in Git Bash (or any bash-compatible shell):

```bash
bash build.sh
```

This configures a Release build with Visual Studio 2022 (x64) into the `build/` directory and copies all required DLLs next to the executable.

To build manually with CMake:

```bash
cmake -S . -B build -G "Visual Studio 17 2022" -A x64
cmake --build build --config Release
```

The output executable will be at `build/Release/VideoPlayer.exe`.

### Running

To play the included test video:

```bash
bash run.sh
```

To play any other video file:

```bash
./build/Release/VideoPlayer.exe path/to/your/video.mp4
```

## Controls

| Input | Action |
|---|---|
| `Space` | Play / Pause |
| Click progress bar | Seek to position |
| `0.5x` / `1x` / `1.5x` / `2x` buttons | Change playback speed |
| Volume icon | Toggle mute |
| `F` or fullscreen button | Toggle fullscreen |
| Move mouse | Show control bar (auto-hides after 3 seconds) |

## Architecture

The player is split across five source files:

| File | Responsibility |
|---|---|
| `main.cpp` | Entry point, argument validation |
| `VideoPlayer.cpp` | Orchestrates playback loop across two threads |
| `Decoder.cpp` | FFmpeg decoding and audio resampling |
| `Renderer.cpp` | SDL3 window, YUV rendering, audio output |
| `PlayerUI.cpp` | Control bar UI, hit-testing, auto-hide logic |

Playback is synchronized to the audio clock with latency compensation. The main thread pumps OS window events while a dedicated render thread owns SDL and handles all drawing.
